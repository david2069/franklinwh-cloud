# Thank You 🙏

A special thanks to **[Richo](https://github.com/richo)** and his [franklinwh-python](https://github.com/richo/franklinwh-python) library and [homeassistant-franklinwh](https://github.com/richo/homeassistant-franklinwh) HACS integration — this library would not exist without them.

---

## The Foundation

| Project | Description |
|---------|-------------|
| [franklinwh-python](https://github.com/richo/franklinwh-python) | The original Python library for FranklinWH cloud API access — authentication, power flow data, and core data models ([contributors](https://github.com/richo/franklinwh-python/graphs/contributors)) |
| [homeassistant-franklinwh](https://github.com/richo/homeassistant-franklinwh) | Home Assistant HACS integration that brought FranklinWH systems into the HA ecosystem ([contributors](https://github.com/richo/homeassistant-franklinwh/graphs/contributors)) |

## What They Made Possible

Richo's work established the critical first pieces:

- **Authentication flow** — reverse-engineering the FranklinWH cloud login and token refresh
- **Core data models** — `Stats`, `Current`, `Totals` structures for power flow data
- **Home Assistant integration** — proving that third-party access to FranklinWH systems was viable
- **Community momentum** — showing that an open-source approach to energy system integration was worth pursuing

This library extends that foundation with 70+ API methods, TOU scheduling, power control, device discovery, and more — but none of it would have been possible without the trailblazing work that came first.
